# 📖 Quick Start Guide

Welcome to your Burp Suite & Web Security Learning Portfolio! This guide will help you get started.

## 🚀 Getting Started

### First Steps

1. **Update Your Information**
   - Edit `README.md` and replace placeholder links with your actual social media profiles
   - Update the LinkedIn and Twitter badge URLs
   - Add your actual LinkedIn profile URL

2. **Configure Your Repository**
   - Follow the instructions in `.github/PORTFOLIO_SETUP.md`
   - Add repository description and topics
   - Consider enabling GitHub Pages

3. **Start Learning!**
   - Choose a topic from the Learning Path in `README.md`
   - Create your first lab in the `labs/` directory
   - Take notes in the `notes/` directory

## 📚 How to Use This Portfolio

### Adding New Documentation

1. Navigate to `docs/` directory
2. Copy `topic-template.md`
3. Rename it to your topic (e.g., `sql-injection.md`)
4. Fill in the sections as you learn
5. Update `docs/README.md` to link to your new document
6. Commit your changes

### Creating a New Lab

1. Navigate to `labs/` directory
2. Copy `lab-template.md`
3. Rename it (e.g., `01-burp-setup.md`)
4. Follow the template structure
5. Add screenshots and evidence
6. Update `labs/README.md` with your new lab
7. Mark it as complete in the main `README.md` progress tracker
8. Commit your changes

### Taking Notes

1. Navigate to `notes/` directory
2. Create a new markdown file (e.g., `2026-02-week1.md`)
3. Document what you learned
4. Link to related resources
5. Commit regularly

### Finding Resources

- Check `resources/README.md` for curated links
- Add new resources as you discover them
- Organize by category

## 💡 Tips for Success

### Documentation
- Write as if teaching someone else
- Include screenshots and examples
- Reference your sources
- Keep it simple and clear

### Labs
- Document every step
- Note what worked and what didn't
- Include error messages and solutions
- Add time spent and difficulty level

### Consistency
- Set a learning schedule
- Commit regularly (even small updates)
- Review and revise earlier content
- Celebrate milestones!

## 🆘 Common Questions

### How often should I update?
As often as you're learning! Even small updates show progress. Aim for at least weekly commits.

### What if I make mistakes?
Document them! Mistakes are valuable learning experiences. Create a troubleshooting section in your labs.

### Should I document everything?
Focus on key insights and practical knowledge. Quality over quantity.

### How do I share this portfolio?
- Pin it to your GitHub profile
- Link it on your LinkedIn
- Include it in your resume
- Share it on Twitter/professional networks
